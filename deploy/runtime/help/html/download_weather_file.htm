Download Weather File

SAM's Download Weather File feature allows you to type an address, zip code, or latitude and longitude to download a weather file from either the NREL National Solar Radiation Database (NSRDB) or NREL WIND Toolkit databases:

•Solar Resource Data

•Wind Resource Data

Notes.
 
SAM's Download Weather File feature uses the Google Maps API Geocoding Service service to translate a street address into a latitude and longitude.
 
For information about downloading weather files and data from other sources, or for locations outside the United States, see the Weather Data page on the SAM website.

To download a weather file:

1.Depending on the performance model you are using, go to the Location and Resource, Wind Resource, Location and Ambient Conditions, or Ambient Conditions page, depending on the performance model you are using.

2.Click the Download button and enter information about the project location and type of data you want to download. Click Help for more detailed instructions.

3.Click OK to download the file.

Solar Resource Data

For the following performance models, SAM downloads weather data in the SAM CSV format from NREL's National Solar Radiation Database (NSRDB):

•Photovoltaic

•Concentrating solar power: Parabolic trough, power tower, linear Fresnel, dish Stirling, and CSP generic solar.

•Solar water heating

•Biomass Combustion

•Geothermal

The NREL National Solar Radiation Database (NSRDB) allows you to download solar resource data in the SAM CSV format. Files are available both as typical-year files and single-year files for 1998 through 2014. The database covers the parts of the Americas and South Asia shown in the map below. The current data is called PSM (Physical Solar Model) for the U.S., and SUNY for Asia. For details about the data see these pages on the NSRDB website:

•U.S. data

•International data

The NSRDB also contains legacy datasets for the U.S. from older versions of the NSRDB, including MTS3 (TMY3 1991-2010), MTS2 (TMY3 1991-2005), and MTS1 (TMY2 1961-1990). We recommend that you use the latest PSM data for your work in SAM, but the older data is available if you need it.

Important Note for CSP Modelers! SAM's CSP models assume that each DNI value in the weather file represents the average irradiance over the time step, and calculate sun angles for the midpoint of the time step. In weather files from the NSRDB, the irradiance data is measured instantaneously at either the beginning of the hour (TMY data), midpoint of the hour (single-year data), or beginning of the 30-minute time step (30-minute single-year data). As a result, if you use an NSRDB weather file with a CSP model with either hourly TMY data or 30-minute single-year data, the sun angle for each time step will be for a different time than the time for the instantaneous irradiance values.

 

Notes.
 
The NSRDB requires an email address to download weather file. When you download a file from SAM, SAM submits the email address that you used to register SAM. NREL uses your email address to determine how many people are using the NSRDB. It does not share email addresses.
 
Different types of weather files in the NSRDB use different time conventions. See Time Convention and Sun Position for details.
 
When you download a file from the NSRDB website, SAM displays a "-" for the City, State, and Country fields because the NSRDB files do not contain that information. This will not affect simulation results because SAM only uses latitude, longitude, elevation and time zone information during simulation, and that information is included in the files.
 
SAM uses the Google Maps API Geocoding Service service to identify the geographic coordinates of a location when you type a street address or city and state name to identify the location.
 
For information about downloading weather files and data from other sources, see the Weather Data page on the SAM website.

SAM downloads weather files with the following hourly data from the NSRDB:

•Global horizontal irradiance (W/m²)

•Direct normal (beam) irradiance (W/m²)

•Diffuse horizontal irradiance (W/m²)

•Dry-bulb temperature (°C)

•Dew-point temperature (°C)

•Relative humidity (%)

•Atmospheric pressure (mbar)

•Wind speed (m/s)

•Wind direction (°E of N)

 

Option 1: Download a TMY file for Americas

If you are using SAM to predict the performance of a system over a lifetime of many years or to calculate financial metrics based on a project cash flow, and your project is in the United States, Canada, Central, or South America, use this option to download a typical-year (TMY) file from the NSRDB.

To download a typical-year (TMY) weather file from the NSRDB:

1.Before you download a weather file, click Folder Settings, and choose or create a folder on your computer to store downloaded weather files. You do not need to do this every time you download a file.

2.Click Download a weather file for your location to open the Download window.

3.Type a street address, zip code, or latitude and longitude in the search box. For example, any of the following four lines will return results for a location in Golden, Colorado:

golden colorado
15013 Denver West Parkway, Golden CO
80401
39.75, -105.15

4.Click OK to download the file. It may take a few moments for SAM to connect to the NSRDB servers and to add the file to the solar resource library.

Note. An error message like "The 'wkt' parameter did not match any known data points" means that there is no data in the NSRDB for the location you specified.

5.When the download is complete, SAM adds the file to your solar resource library and automatically selects it. You can see a summary of the data in teh file on the Location and Resource page, and click View weather file data to see graphs of the data.

Option 2: TMY or Single-year for Americas and Asia

Use this option to download the following types of files from the NSRDB:

•Typical-year and single-year PSM files for the U.S., Mexico, Caribbean, Central America, and northern South America. The single-year files are those used to construct the PSM TMY files. You can download a single file for a year between 1998 and 2014 if you want to model the performance of a system in one of those years, or download all of the files for a P50/P90 analysis. The PSM files contain the current up-to-date data from the NSRDB.

•Typical-year and single-year SUNY files for South Asia, and parts of Pakistan and Afghanistan.

•Legacy MTS1, MTS2, or MTS 3 typical-year files for the U.S. These are the old TMY2 and TMY3 files from the NSRDB. Use these files to compare results from analyses using the older data sets or if you have have a reason for using the older files.

Note. For a list of weather file sources for locations not included in the NSRDB, see the Weather Data page on the SAM website: https://sam.nrel.gov/weather.

To choose files to download from the NSRDB:

1.Click Choose files to download (advanced).

2.Type a street address, zip code, or latitude and longitude in the Find location box. For example, any of the following four lines will return results for a location in Golden, Colorado:

golden colorado
15013 Denver West Parkway, Golden CO
80401
39.75, -105.15

3.Click Search.

4.In the list of files that appears, choose the files you want to download. Choose PSM 60-minute data for most analysis. See the description and links to NSRDB website above for more information about the types of files.

To filter the list, type a few characters in the Search box. For example, type "mts2" to show only MTS2 files. You can click the Select PSM hourly, Select PSM 30-minute buttons to filter the list for those files.

5.Click Select filtered to check only the files in the filtered list.

Note. If you are downloading single-year files, you may need to clear checkbox for the TMY file to avoid downloading it.

6.Choose a download folder to store the files. SAM will automatically add this folder to the list of solar data file folders.

7.Click OK to start the download. SAM will connect with the NSRDB, download the files you chose, and add the files to your solar resource library.

Wind Resource Data

NREL's Wind Integration National Dataset (WIND) Toolkit provides wind speed and direction, ambient temperature, and atmospheric pressure at 100 m above the ground data for 126,692 sites in the continental United States and parts of Central America and the Caribbean. Of those sites, 112,471 are on land, and 14,221 are offshore in coastal areas. The WIND Toolkit provides historical single-year data for the time period between 2007 and 2012.

For a description of the WIND Toolkit:

•Wind Integration National Dataset Toolkit website

•Draxl, C et. al. Advancements in Wind Integration Study Data Modeling: The Wind Integration National Dataset (WIND) Toolkit. Preprint. 2013. NREL Conference Paper. NREL/CP-5D00-60269. (PDF 142 KB)

For an interactive map showing the site locations:

•Wind Prospector website

The WIND Toolkit API on the NREL Developer Network:

•WIND Toolkit API

Map from Wind Prospector website showing locations of WIND Toolkit data sites. Colors indicate the capacity factor for each site as estimated by the toolkit.

Each wind data site is represented in the WIND Toolkit as a 2 km by 2 km grid cell. When you use SAM to request wind data for a project location, SAM sends the location's latitude and longitude to the toolkit. The toolkit finds the wind data grid cell nearest the project location and returns a weather file that SAM stores on your computer. SAM names the file using the latitude and longitude of the project location you provided. You can find the latitude and longitude of the wind data grid cell in the header of the weather file. When you use a street address in SAM to describe the project location, SAM translates the street address to a latitude and longitude using the Google Maps Geolocation API.

To download a weather file from the NREL WIND Toolkit:

1.Click Download.

2.Type a latitude and longitude, street address, or zip code.

3.Choose a year.

4.Click OK.

SAM displays a message when the download is complete, and checks the box under Use a wind resource file stored on your computer and shows the path of the downloaded file.
